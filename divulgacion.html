Portal Académico
Proyecto GIISI - USAL
Requisitos Iniciales y Funcionales

   Se desea diseñar una aplicación web para la Gestión Colaborativa e Intercambio de Material Académico.

   La aplicación será un portal web facilitador donde estudiantes y docentes podrán compartir apuntes, exámenes y prácticas. La solución prioriza la localización rápida de recursos y la calidad mediante un sistema de validación y reputación (comentarios y valoraciones).

   Se podrán realizar búsquedas por el repositorio de documentos mediante filtros por categoría (asignaturas, cursos), tipos de documento (apuntes, tesis, guías) y autores. Los resultados mostrarán listados paginados de recursos disponibles para su consulta o descarga.
Requerimientos para el Público en general:
  • Portada "Inicial": Una página de aterrizaje (Landing Page) que explique el propósito del portal, con una barra de navegación global (Inicio, Materiales, Dudas, Login, Registro). Debe incluir un buscador central prominente para localizar materiales por título o palabra clave.

  • Catálogo de Materiales (Lectura): Acceso al listado de documentos públicos. Se permitirá filtrar por categorías jerárquicas (Ej: Grado > Curso > Asignatura).
    • Los listados mostrarán una tarjeta resumen con: Título, Tipo de archivo, Autor y Valoración media.

  • Ficha de Documento (Detalle): Visualización de la información completa del recurso: Título, descripción/resumen, fecha de publicación, autor, categoría asociada, tamaño del archivo y lista de comentarios/valoraciones de otros usuarios.

Requerimientos para Usuarios Registrados (Roles: Normal y Colaborador):
  • Perfil de Usuario: Gestión de datos personales (nombre, email, avatar). Capacidad para ver su historial de descargas o interacciones.

  • Interacción con Materiales:
    • Descarga: Permiso para descargar los archivos adjuntos (PDF, DOC, ZIP).
    • Valoración y Comentarios: Posibilidad de puntuar recursos (1-5 estrellas) y añadir reseñas textuales para orientar a otros estudiantes.
    • Dudas y Respuestas: Acceso al foro de preguntas donde pueden plantear dudas sobre materias o responder a las de sus compañeros.

  • Funciones de Colaborador (Usuario Avanzado):
    • Subida de Documentos: Formulario para aportar nuevo material. Requiere introducir metadatos (título, resumen, categoría) y adjuntar el fichero. Estos documentos quedarán en estado "Pendiente" hasta su validación.
    • Gestión de Colecciones: Capacidad para agrupar varios documentos relacionados (ej: "Pack de Apuntes 1º Curso") en una entidad "Colección" para facilitar su difusión conjunta.

Requerimientos para Administradores:
  • Moderación de Contenidos: Listado de documentos pendientes de validación. Acciones permitidas:
    • Publicar: Hace visible el documento en el portal.
    • Rechazar: Devuelve el documento al autor con un motivo de rechazo (campo de texto).

  • Gestión de Entidades Maestras:
    • Usuarios: Bloqueo/desbloqueo de cuentas, cambio de roles.
    • Categorías: CRUD (Crear, Leer, Actualizar, Borrar) de las categorías y asignaturas del sistema.
    • Comentarios: Moderación y eliminación de comentarios inapropiados.

Información (Entidades de Datos):
A continuación se describe la estructura de información basada en el modelo de datos definido:

  • Usuario: Elemento con → id, nombre, email, contraseña (hash), rol (Normal/Colaborador/Admin), fecha_registro, estado (activo/inactivo).

  • Documento: Elemento con → id, título, descripción/resumen, nombre_fichero, ruta_archivo, fecha_publicacion, estado (pendiente/publicado/rechazado), id_autor, id_categoria.

  • Categoría: Elemento con → id, nombre, descripción, id_padre (para jerarquías). Permite organizar los contenidos por Grados o Asignaturas.

  • Comentario / Valoración: Elemento con → id, texto_comentario, puntuación (numérica), fecha, id_usuario, id_documento.

  • Colección: Elemento con → id, nombre, descripción, fecha_creacion, id_propietario.
    • Relación Colección-Documento (N-M): Permite que un documento pertenezca a varias colecciones temáticas.

  • Duda / Respuesta: Elemento con → id, título, cuerpo, fecha, id_autor. Las respuestas se vinculan a una duda principal.
